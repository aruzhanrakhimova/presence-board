# PRD: Табло присутствия сотрудников

## 1. Проблема

При согласовании и утверждении документов в СЭД (системе электронного документооборота) документ может быть направлен сотруднику, который в этот момент находится в отпуске, на больничном, в командировке или отсутствует по другой причине. Это приводит к простою документа в очереди согласования, срыву сроков и необходимости вручную выяснять, кто сейчас доступен.

## 2. Цель продукта

Дать сотруднику, формирующему маршрут согласования в СЭД, простой способ за секунды проверить, кто из участников доступен сегодня, а кого лучше не указывать в маршруте — до того, как документ отправлен.

## 3. Целевые пользователи

- Делопроизводители и секретари, формирующие маршруты согласования.
- Руководители подразделений, назначающие согласующих.
- Сами сотрудники — самостоятельно отмечают свои периоды отсутствия.

## 4. Пользовательские сценарии

1. **Проверка перед отправкой документа.** Пользователь открывает приложение, ищет сотрудника по имени и видит его текущий статус (на месте / недоступен, с указанием причины и даты возвращения).
2. **Обзор дня.** Пользователь открывает приложение и сразу на верхнем табло видит, сколько человек доступны, а сколько отсутствуют, без поиска по каждому.
3. **Внесение отпуска/больничного.** Сотрудник или HR добавляет период отсутствия: тип, даты, комментарий.
4. **Уведомление коллег.** Пользователь одной кнопкой формирует сводку отсутствующих и отправляет её на WhatsApp, чтобы коллеги, формирующие маршруты, были в курсе.
5. **Ведение справочника.** Добавление новых сотрудников, редактирование и удаление записей.

## 5. Функциональные требования

### 5.1 Табло статуса (шапка)
- Текущая дата.
- Счётчик «доступны» и «отсутствуют» на сегодня, обновляется автоматически.
- Кнопка «Отправить сводку на WhatsApp» — формирует текстовый список всех отсутствующих сегодня (ФИО, отдел, тип отсутствия, дата окончания) и открывает WhatsApp-чат с номером получателя с готовым текстом.

### 5.2 Список сотрудников
- ФИО, отдел/должность.
- Индикатор текущего статуса: На месте / Отпуск / Больничный / Командировка / Удалённо / Другое.
- Дата, до которой сотрудник недоступен (если применимо).
- Поиск по имени и отделу.
- Фильтры: Все / Доступны / Отсутствуют / по конкретному типу отсутствия.

### 5.3 Карточка сотрудника (разворачивается по клику)
- История периодов отсутствия (тип, даты, комментарий).
- Добавление нового периода отсутствия: тип, дата начала, дата окончания, комментарий.
- Удаление отдельного периода отсутствия.
- Точечное уведомление на WhatsApp по конкретному периоду отсутствия.
- Удаление сотрудника из справочника.

### 5.4 Добавление сотрудника
- Форма: ФИО (обязательно), отдел/должность (опционально).

### 5.5 Логика доступности
Сотрудник считается **доступным**, если у него нет активного периода отсутствия на сегодняшнюю дату, либо активный период имеет тип «Удалённо» (удалённая работа не блокирует согласование).
Типы, делающие сотрудника **недоступным**: Отпуск, Больничный, Командировка, Другое.

*(Правило настраиваемое — при необходимости соответствие «тип → доступность» можно изменить.)*

### 5.6 Уведомления WhatsApp
- Ручная отправка: приложение формирует текст сообщения и открывает `wa.me`-ссылку с номером получателя и готовым текстом; пользователь подтверждает отправку в интерфейсе WhatsApp.
- Автоматическая фоновая отправка не реализована (требует backend и интеграцию с WhatsApp Business API — вне рамок текущей версии).

## 6. Модель данных

Общий (shared) объект `roster` — массив сотрудников:

```json
{
  "id": "string",
  "name": "string",
  "department": "string",
  "absences": [
    {
      "id": "string",
      "type": "vacation | sick | trip | remote | other",
      "from": "YYYY-MM-DD",
      "to": "YYYY-MM-DD",
      "comment": "string"
    }
  ]
}
```

Данные общие для всех, кто открывает приложение: любой пользователь может добавлять сотрудников, отмечать и удалять периоды отсутствия.

## 7. Нефункциональные требования

- Все данные хранятся и обновляются в реальном времени для всех пользователей приложения (общее хранилище).
- Интерфейс адаптивен, работает на мобильных и десктопных экранах.
- Не требует установки — работает как веб-приложение (артефакт).

## 8. Вне рамок текущей версии

- Автоматическая фоновая рассылка уведомлений (push/по расписанию) без действия пользователя.
- Импорт сотрудников из Excel/1С.
- Ролевая модель доступа (например, только HR может добавлять отсутствия).
- Согласование дат отсутствия (двойное подтверждение).
- Интеграция напрямую с СЭД (автоматическая блокировка выбора отсутствующего согласующего в самой системе документооборота).

## 9. Возможные развития продукта

- Импорт списка сотрудников из Excel.
- Недельный/месячный календарный вид отсутствий.
- Уведомление о скором возвращении сотрудника.
- Прямая интеграция с СЭД через API (автоматическая проверка при выборе согласующего).
- Настраиваемые правила доступности по типам отсутствия и по отделам.
